1.Overview

From the user's point of view, the application would present a dynamic, navigational 3D space, that allows free control and interaction of objects while providing a basic platform that ensures privacy, trust, and security.

The space is created dynamically by it's peers, which does not allow any central entity. Its platform, though, implements certain rules and rights for users, i.e. 'virtual free speech' and 'virtual property' laws. Thus, the space promotes equality and freedom. The objects could be an environment, a resource, a function, bots, a service, or a link to other existing (or legacy) infrastructures, such as the web, X3D or VRML.

By focusing on the balance of keeping the space free, while not anarchistic, the main purpose of an implementation would be to create a dynamic distributed infrastructure where user's could roam, meet people, build private or public places, provide services, and search resources.

The project implements a distributed (P2P) application, that...

  • can connect ad-hoc to the de-centralized peer-network
  • a dynamic space that visualizes the peer-network in 3D
  • user representation by avatars
  • provides navigation control interfaces
    • coordinate-based world
    • level-based transport routes
    • 3D or peer-based bots
    • agent-based (autonomous) search, seek & retrieve
    • object interaction, events
    • mechanisms to cope with user attention
  • provides logic to cope with privacy, trust, and security
    • basic user rights and space laws/restrictions
    • space navigation allowance, reservation/expiration, and settlement (zoning)
    • object control (ownership, read/view, create, modify, and delete)
    • space and object sharing
  • is customizable (personalization)
  • could store and handle personal information (privacy)
    • security systems that support authentication, integrity, and confidentiality
    • set permissions for sites how to use your data
    • set requirements for user-data for your own site
    • store user info and provide data-mining tools
  • could provide a standard interface for plug-ins, which can host a variety of platforms, such as
    • X3D/VRML spaces and IDE's
    • avatar and bot creators
    • personal and business web-sites/-services
    • peer-trading
    • a trust-creating feedback system
    • resource sharing (file/document sharing)
    • autonomous agent-based trading
    • communication tools (messengers, web-logs)
    • data-mining tools and knowledge management

4.Space

The space could be build up using a static 3D grid that spawns in a x/y-direction with a specific detail, and several layers in the z-direction. A center origo could provide as a starting point and navigational 'well-known-location'. Navigational aids, such as static directions and coordinates could be provided by certain conventions (North, South, IPv6 address).

The z-layers would provide different levels of virtual transportation, which could allow both restricted and free space.

The current view would depend on the peers knowledge of the space, the connectivity with other peers, spatial and visibility limitations, and the current position of the user's avatar.

Visualization aids, such as area coloring, the unlimited origo axis, a horizon, and visual details (or lack thereof) could enhance usability. Space, navigational aids, avatars, bots, sites and objects could thus be displayed dynamically and by choice.

Global space rights and restrictions could enable open creation, secure individual freedom, privacy, and trust, while preventing anarchy. Object control would support this as well. Virtual property could be established, reserved, traded, and connected. Restrictions on each peer's virtual space (borders) could enable full personalization, including access control, interplay with connecting land, 'physical' laws, and functionality inside that space.

3D rendering, collision detection, search algorithms, and networking are all processing-intensive tasks that have to be minimized in an implementation. The distributed architecture combined with dynamic visualization could prove useful in that matter.
